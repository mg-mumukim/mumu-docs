# Note

Source material organized by scope.

File naming: `yyyy-MM-dd-[a-z0-9-].md` (e.g. `2026-04-03-auth-spec.md`)

## Structure

```
note/<scope>/    source material for that scope
note/images/     shared images referenced by source files
```

Each `<scope>` directory corresponds to the `<scope>` prefix used in `work/<scope>-<subject>/`. One scope directory in `note/` feeds one or more deliverable directories in `work/`.

| Scope | Description |
|-------|-------------|
| `photo-review` | Photo review feature (cost estimation, privacy, serving options) |
| `photo-suppression-v2` | Photo suppression v2 experiment |
| `aura` | Aura AI product |
| `dlp` | Data loss prevention system |
| `opaas` | OPaaS sunset |
| `smartphoto-v4` | SmartPhoto v4 engineering |
| `agent-skill` | Claude agent skill research and practice |
| `reference` | Cross-cutting reference material not tied to a specific project scope (playbooks, skill guides, methodology documents) |
