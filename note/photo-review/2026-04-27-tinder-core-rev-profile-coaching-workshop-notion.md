# Tinder Core x Rev Profile Coaching Workshop

## Background

- Tinder 제품 전반에서 "코칭"이 어떤 역할을 해야 하는지 정의하고, 핵심 사용자 문제와 기회를 정리하기 위한 자리
- 특히 유저 여정 전반에서 코칭이 가장 큰 가치를 만들 수 있는 지점을 찾고, 어떤 기능이 기본 제공인지 vs 유료 차별화 요소인지 기준을 세우는 것이 목적
- 여러 팀이 각각 코칭 기능을 만들고 있는 상황에서, 개별 기능이 아니라 하나의 일관된 제품 방향과 원칙을 정리하고자 함

## Summary

### S1. Research Shareout 요약

- 코칭이 가장 직접적으로 해결할 수 있는 핵심 문제는 "좋은 사람을 못 만난다"이며, 이는 프로필 품질과 강하게 연결되어 있다.
- 여성 유저는 프로필 퀄리티를 "존중의 신호"로 해석하며, 성의 없는 프로필은 곧 나를 존중하지 않을 사람으로 인식된다.
- 남성 저성과(RSRR 낮은) 유저는 자신의 프로필 문제를 인지하지 못하거나, ecosystem의 구조적 문제라고 굳게 믿어 코칭에 대한 저항이 크다.
- 유저는 자기가 이미 프로필 구성에 대해 나름의 노력을 하고 있다고 생각하기 때문에, 단순한 팁이 아니라 "왜 이게 문제인지"에 대한 설명이 동반되어야 한다.
- 코칭은 한 번의 개선으로 끝나는 것이 아니라, 반복적이고 점진적인 개선 구조로 설계되어야 한다.
- 현재 시장의 코칭은 대부분 표면적인 추천 수준에 머물러 있으며, 개인화된 인사이트와 실제 행동 변화까지 이어지는 경험은 부족하다.
- 코칭의 가치는 "얼마나 대신 해주느냐"와 "얼마나 나에게 맞게 느껴지느냐"에서 결정된다.
- 프로필 개선에서 사진의 영향력이 가장 크며, 기본적인 베이스라인(선명한 얼굴, 다양한 사진 등)을 먼저 해결하는 것이 중요하다.
- AI 코칭은 선택권을 사용자에게 남기고, 이유가 설명되며, 결과 개선이 명확할 때에만 신뢰를 얻는다.
- 코칭은 수익화와 강하게 붙이면 불신을 유발할 수 있어, placement와 framing이 신뢰 형성에 핵심적이다.

### S2. Define Coaching

- 코칭의 핵심 역할은 단순 팁 제공이 아니라, "왜 지금 결과가 나오고 있는지"를 데이터 기반으로 설명하는 것
- 가장 공감된 use case는 "내 성과를 제한하는 요인을 알고 싶다"였으며, 이는 자기 인식 부족 문제와 직결됨
- 코칭은 프로필을 넘어서 스와이프, 행동 패턴, 의도 vs 실제 행동의 불일치까지 확장되어야 한다는 방향
- 사용자에게 판단받는 느낌 없이, 신뢰 가능하고 비판단적인 톤으로 전달하는 것이 중요
- 주요 tension은 "넓은 매력(broader appeal)" vs "진정성/구체성(authenticity)"
- 현재는 프로필 수준이 전반적으로 낮기 때문에, 1차 목표는 넓은 매력으로 끌어올리는 것이 더 중요하다는 의견
- 특정 취향은 숨기기보다, 더 다양한 맥락 속에서 균형 있게 드러내도록 코칭하는 방향
- 과도한 최적화나 자동 생성은 신뢰를 해칠 수 있어, 사용자 참여(agency)가 중요
- 전반적으로 "데이터 기반 인사이트 + 사용자 선택 유지"가 코칭 설계의 핵심 방향으로 도출됨

### S3. Free vs. Paid Framework

- 기본적인 프로필 피드백(인상, 베스트 프랙티스)은 무료에 두고, 초기 진입 및 업셀 진입점으로 활용하는 방향
- 코칭은 최소 기준까지는 무료, 그 이상의 디테일/고도화는 유료로 나누는 layered 접근이 유력
- 지속적인 코칭은 무료 vs 유료 의견이 갈렸지만, 기본적인 개선 루프는 무료에 두는 쪽에 무게
- 무료는 "매칭이 발생하도록 돕는 영역", 유료는 "더 좋은/맞는 사람과 연결되는 영역"으로 구분
- 개인화는 전부 유료로 둘 수 없으며, 무료는 기본 인사이트, 유료는 깊은 행동 기반 개인화로 확장
- "알려주는 코칭"은 무료, "자동으로 바꿔주는 코칭"은 유료/옵트인에 가깝다는 구분
- 데이터 인사이트는 반드시 행동으로 이어져야 하며, 과도한 정보 제공은 오히려 역효과 가능
- 성과 퍼널(관심→매치→대화→만남) 기준으로, 상위 단계는 무료, 하위/깊은 단계는 유료화 가능
- 코칭을 유료와 강하게 결합하면 신뢰를 해칠 수 있어, trust가 가장 중요한 가드레일
- 유료의 핵심 가치는 "더 깊은 개인화 + 더 나은 매칭 + 재미있는 데이터 경험"으로 정리됨

### Pre-read 내용 요약

**User Problems** — 3가지 핵심 불만:

| 문제 | 설명 |
|------|------|
| "마음에 드는 사람을 잘 못 만난다." | 외모뿐 아니라 진짜 사람 같다는 느낌, 존중, 평판과 연결됨 |
| "앱에 쓴 시간만큼 보람이 없다." | 매치 → 대화 → 연락처 교환 → 실제 만남까지 진도가 나가는지로 측정 |
| "틴더가 부담스럽다." | 나 자신, 상대 유저, 앱이 어떻게 돌아가는지에 대한 불확실함에서 비롯 |

세대·그룹별 특이사항:
- Gen Z: 속도만이 아니라 마음이 편해야 함. 상대방의 거절이 두렵고, 솔직한 연결·친구처럼 가벼운 시작 선호
- 남성: 매치와 대화가 더 생기길 바라고, 그것 때문에 결제를 고려
- 여성: 내게 맞는 사람, 안전함, 통제감에 민감. 무례한 첫 메시지나 대충 만든 프로필에 쉽게 마음을 닫음

**프로필 만드는 방식** — 3가지 유형:

| 유형 | 설명 |
|------|------|
| 일부러 텅 비워 둔 타입 | "노력 안 한 듯 멋있어 보이기" 또는 덜 진지해 보이려고 최소만 채움 |
| 한 주제에만 몰린 타입 | 사진/글은 많은데 스포츠/차/술 같은 좁은 주제만 반복 |
| 조언을 잔뜩 따라 하다 정리 안 된 타입 | 여기저기서 들은 팁 적용하다 기본 내용이 빠지거나 산만해진 경우 |

코칭 설계 방향: 객관적으로 알려주고 / 왜 그런지 설명하고 / 최종 선택은 사용자에게 남기는 구조

**프로필 항목별 중요도**:
- 사진 매력도가 한 단계 오르면 bio보다 프로필 전체 인상에 미치는 영향이 훨씬 큼
- 첫 장 사진은 얼굴이 선명하게, 여러 장, 서로 다른 삶의 모습 최소 두 가지 이상
- 바이오는 있는 것 자체가 중요, 두세 문장에 본인 특징·무엇을 찾는지·유머가 들어가면 "진짜 사람이 노력했다"는 신호

**경쟁사 분석**:
- 코칭 스펙트럼: 신호 포착 → 설명 → 제안 → 구체화 → 대신 실행 → 습관화
- 많은 제품은 "이렇게 해보세요" 수준 또는 앞 단계 없이 추천만 함
- 개인화되지 않은 팁은 "코칭 같지 않다"는 인상

**AI 코칭이 받아들여지는 조건**:
- 노력을 줄여주거나 결과가 분명히 나아질 때
- 제안만 하고 선택은 사용자가 할 때
- 왜 그런 조언인지 이유가 보일 때
- 취향을 해치지 않을 때

**코칭이 붙을 만한 지점 (JTBDs)**:

| 방향 | 한 줄로 |
|------|---------|
| 좋은 사람/경험 | 나한테 보여지는 사람이 진짜로 보이고, 나를 존중함 |
| 시간이 아깝지 않게 | 매치부터 만남까지 진행되는 것이 느껴지고 보이게 |
| 덜 부담스럽게 | 나 자신, 상대, 앱에 대한 막연함 줄이기 |
| 프로필 개선 | 사진/글을 근거 있게 다듬고, 나만의 이야기가 드러나게 하기 |
| 믿을 만한 도움 | 왜 하라는건지 이유가 보이고, 선택권이 있고, 취향을 존중하는 AI/가이드 |
| 대화/만남 | 모멘텀, 무례한 첫 메시지, 진정성 — AI가 얼마나 대신 해주는지에 민감 |
| 유저를 "더 나은 Dater"로 만들기 | 현재는 out of scope로 보고 있음 |

- AI 대행이 늘면 진정성 인식이 떨어질 수 있어 개인화된 인사이트가 중요하다는 의견에 모두 합의

### Jeremy — 유저 문제 정리 + 저RSRR 남성 심층 리서치 (A2)

**공통 유저 문제(코칭 연결 우선순위)**:
1. 틴더에서 좋은 사람을 못 만난다 — 코칭과 가장 잘 맞는 축
2. 압박이 크다 — 두 번째로 중요
3. 시간 대비 가치 — 1번과 많이 겹쳐서 별도 과제로 두진 않을 것

**프로필 품질에 대한 남녀 차이**:
- 남성: 이 사람이 얼마나 매력적인지 보기 전에 진짜 사람 같이 느껴지는지(realness)를 봄
- 여성: 프로필이 성의 없으면 나를 존중하지 않는 사람일 것 같다는 인식 → 스와이프 멈춤

**LOW RSRR 남성 연구** (미국, 최하위 분위 남성 9명 심층 인터뷰):
- 공통점: 본인 프로필이 나쁘다고 생각하지 않음, 본인 사진에 대체로 만족
- "실패는 틴더에서 남자들한테 흔한 경험"이라는 서사 → 도움이 굳이 필요 없다는 쪽으로 기울기 쉬움
- 결제 의향 낮음: 틴더가 성공시키려는 게 아니라 돈 벌려는 구조라는 불신이 큼

**저RSRR 남성 3가지 프로필 유형**:
1. 최소만 올린 사람 — "은근 멋있어 보이려고", 덜 진지해 보이려고. 이 그룹이 가장 많음
2. 자기표현에 몰두한 사람 — 분량은 많은데 한쪽 주제만 반복, bio를 정치 필터처럼 사용
3. 이것저것 온라인/지인 팁을 따라 한 사람 — 열심히 했는데 기본이 빠진 경우

**유형별 "성공"의 의미**:
- 최소 그룹: 양, 캐주얼한 티
- 자기표현 그룹: 나 자신을 좋아해 주는 사람
- 호환 전략 그룹: 의미 있는 연결

**피드백 저항**: 보통 팁을 주면 "남한테 좋은 조언이지 나한테 해당 안 된다"고 많이 느낌

### 토론에서 오간 이야기 (A3)

- **Growth 실험(Photo Feedback)**: 조언을 보고 실제로 행동한 비율, 사진 교체율, RSRR 상승(여성에서 소폭), 리텐션 개선 관찰. 단, 사진을 바꿨는데 더 안 좋아진 경우도 있어 한 번의 코칭으로 끝내면 안 된다는 인사이트 도출
- **여성 문제 프레이밍**: "좋은 남성이 없다"기보다, 프로필이 성의 없어서 상대를 파악할 수 없고 존중도 기대하기 어렵다는 방향으로 최근 인용이 많아짐
- **Grammarly 비유**: 대부분 유저가 기본 프로필 완성도(선명한 대표 사진 등)조차 부족한 상태라면, 베이스라인을 먼저 끌어올린 뒤 톤/진정성 같은 요소를 다뤄야 함
- **개인화 데이터**: 현재 MVP는 프로필에 드러난 정보를 해석하는 수준. "내가 좋아하는 사람이 선호하는 스타일"까지 반영하는 개인화는 다음 단계. 아직 없는 데이터를 어떻게 수집할지에 대한 아이디어 필요
- **자기 인식 vs 타인 인식**: 사용자들은 자신의 프로필을 평균 6/10으로 평가하면서도 10점짜리가 무엇인지 명확히 모름. 의도한 이미지와 실제로 전달되는 인상이 어긋나는 사례 확인 → 단순 진단만으로도 충분한 가치

## Session 2: Define Coaching

> 코칭의 정의와 제품 내 역할을 정리하고, 해결해야 할 핵심 사용자 문제 2~3개에 대해 합의하는 세션

FigJam: https://www.figma.com/board/P5Yz9V6Y1KjOHRIvn2MgBt/Rev-x-Profile-Coaching-Workshop-Session?node-id=0-1

### 세션 진행 방식

1. 후보 use case 5개에 대해 소그룹 토론
2. 매치 볼륨 vs better fit date 논의 (시간 부족으로 결정 미완)
3. 코칭 프롬프트 및 가드레일 정의

시간이 부족해서 명확한 합의는 도출되지 않음. 진행자가 다음 날 정리본 공유 예정.

### 주요 논의 1 — "내 success를 제한하는 요인을 알고 싶다"

가장 공감된 use case:
- 사용자는 자신의 프로필/행동이 어떤 결과를 만드는지 인사이트가 거의 없음
- Tinder는 데이터를 통해 어떤 요소가 성과와 연결되는지 알고 있음

코칭의 역할:
- 단순한 팁 제공이 아니라 "왜 지금 결과가 나오고 있는지"를 데이터 기반으로 설명

코칭 범위 확장 의견:
- 로그인 빈도, 스와이프 패턴, 행동 vs 의도의 불일치까지

핵심 방향: **결과를 제한하는 요인에 대한 인식 + 행동 기반 자기 인식 + 진정성(말과 행동의 일치)**

### 주요 논의 2 — Specificity/Authenticity vs Broader Appeal

딜레마 예시: "낚시만 좋아하는 프로필" — 너무 구체적이면 좁은 매칭, 너무 일반적이면 진짜 모습이 안 보임

결론 방향:
- 현재 틴더 전체 프로필 수준이 낮기 때문에 1차 목표는 **Broader appeal로 끌어올리는 것**
- 약 80% 문제는 이 접근으로 해결 가능
- 특정 취향은 숨기는 게 아니라 더 넓은 맥락에서 표현하도록 코칭 (낚시 사진 5장 → 1장으로 줄이고 다른 측면 어필)

주요 우려사항:
- 과도한 최적화 → 실제 만남에서 "속았다"는 느낌
- 완전 자동 생성 프로필은 신뢰를 떨어뜨림
- 사용자가 "함께 만든다"는 감각이 중요

### Brainstorming

- 프리매치: 넓은 매력 중심 / 포스트매치: 더 깊은 정보 공개
- 단계적 공개(onion model): 첫인상은 broader appeal 사진만 → 이후 점점 더 구체적인 디테일 공개
- 데이터 기반 투명성: "이딴식으로 프로필 만들면 노출이 좁아진다"를 알려주고 선택은 유저가 하게 하기

| 영역 | 드러난 방향 (합의 아님) |
|------|------------------------|
| 코칭 역할 | 결과를 설명하고 행동 인식을 높이는 데이터 기반 코칭 |
| 사용자 문제 | 자기 인식 부족 + 결과를 막는 요인에 대한 이해 부족 |
| 핵심 tension | 넓은 매력 vs 진정성 |
| 해결 접근 | 단계적 노출, 데이터 기반 설명, 사용자 선택 유지 |
| Tinder 역할 | 대규모 데이터 기반 인사이트 제공 (비판단적 톤 유지) |

## Session 3: Free vs. Paid Framework

> 무료/프리미엄 경험을 나누는 레버는 무엇인지 합의해보는 세션

1. **기본적인 프로필 피드백**: 인상·베스트 프랙티스 수준은 무료. 초기 진입점(후킹)으로 활용 후 업셀 연결. 사진 밝기·선명도 같은 기본 가이드는 생태계 전체 품질을 위해 무료 제공해야 한다는 의견 강함

2. **지속적인 코칭**: 의견 갈림. 최소 기준까지는 무료, 그 이후는 유료. 코칭을 유료와 너무 강하게 묶으면 "돈 벌기 위한 기능"으로 인식될 수 있음

3. **매치 볼륨 vs 매치 품질**: 무료 = 매칭이 일어나도록 돕는 영역 / 유료 = "더 좋은/맞는 사람"과 연결되는 영역

4. **개인화**: 전부 유료로 두면 안 됨. 무료 = 기본 인사이트 / 유료 = 행동 데이터 기반 깊은 분석, 특정 유형에 대한 반응 분석. "알려주는 것" → 무료에 가까움 / "자동으로 바꿔주는 것" → 유료/옵트인에 가까움

5. **데이터 기반 인사이트**: 인사이트만 제공하면 부족 → 행동으로 이어져야 함. 무료 = Tinder 내부 데이터 기반 / 유료 = 더 희귀하거나 비용이 높은 데이터 (예: 상대 피드백)

6. **성과 기반 코칭**: 관심 → 매치 → 대화 → 실제 만남. 상위 퍼널은 무료, 깊은 단계는 유료. 기본 성과도 나오지 않는 상태라면 유료로 해결하게 만드는 것은 부적절

7. **유료 경험의 방향**: 더 깊은 개인화 + 더 좋은 매칭 + 재미 요소(상세 통계, A/B 테스트, 개인 리포트). 숫자만 제공하면 부정적 경험 가능 → 맥락화된 인사이트 필요

8. **신뢰(Trust) 이슈**: 코칭 + 결제를 강하게 연결하면 신뢰 하락. AI가 대신 만든 프로필은 신뢰를 잃기 쉬움. **신뢰가 핵심 가드레일**

**정리된 방향 (합의 아님)**:
- 기본 코칭은 무료로 제공, 최소 기준까지는 무료로 끌어올림
- 유료: 더 깊은 개인화 + 더 나은 매칭 + 더 재미있는 데이터 경험
- 가장 중요한 가드레일: 신뢰를 해치지 않는 방식으로 설계
